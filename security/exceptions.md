# Security Exception Register

Use this register only for temporary risk acceptance when a fix is not yet available.

## Policy

- Exceptions must be time-bound.
- Exceptions require mitigation notes and owner.
- Expired exceptions must fail review.

## Template

| ID | Date | Component | Finding/CVE | Severity | Reason | Compensating Controls | Owner | Expires |
|---|---|---|---|---|---|---|---|---|
| EX-YYYY-001 | YYYY-MM-DD | image/service | CVE-XXXX-YYYY | HIGH/CRITICAL | No upstream fix available | Network restriction + monitoring | team/user | YYYY-MM-DD |
